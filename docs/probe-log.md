# @QuizBot Probe Log

**Date:** 2026-05-20
**Account locale:** Russian
**Method:** Manual quiz creation with 1 question, captured via `probe.py`

## КРИТИЧЕСКОЕ ОТКРЫТИЕ

`@QuizBot` **больше не принимает варианты ответа текстом**. Он ожидает
**нативный Telegram-опрос** (Poll), который пользователь создаёт через UI
Telegram. Внутри poll-а — и вопрос, и варианты ответа.

Это значит:
- Каждый вопрос отправляется **одним Poll-сообщением** через Telethon API
  `client.send_file(InputMediaPoll(...))` или эквивалент.
- После отправки poll-а бот пишет "Пожалуйста, проголосуйте за правильный
  ответ." — нужно проголосовать в собственном опросе через
  `messages.SendVote`, либо сразу отправлять **quiz-poll** с заранее
  установленным `correct_answer`.
- Поле `explanation` из JSON → передаём как `solution` в `InputMediaPoll`
  (поддерживается только для quiz-poll-ов).
- После всех вопросов бот спрашивает **время на ответ** (кнопки) и
  **режим перемешивания** (кнопки) — это новое, в спецификации не учтено.

## Полный транскрипт

### Шаг 1: `/start`

```
TEXT:
С помощью этого бота Вы можете создать **тест** из нескольких вопросов с
**правильными ответами**.

BUTTONS:
  [0,0]: 'Создать новый тест'
  [1,0]: 'Мои тесты'
  [2,0]: 'Язык: Русский'
```

### Шаг 2: клик "Создать новый тест"

```
TEXT:
Вы решили создать новый тест. Для начала, пожалуйста, пришлите название
Вашего теста (например, «Тест на знание математики» или «10 вопросов о
Великой Французской Буржуазной Революции»).
```

### Шаг 3: отправили название теста (текст)

```
TEXT:
Пожалуйста, пришлите описание для Вашего теста. Этот шаг можно пропустить:
/skip
```

### Шаг 4: отправили `/skip` (либо текст описания)

```
TEXT:
Отправьте мне первый вопрос Вашего теста в виде **опроса** с вариантами
ответа. Вы также можете отправить **сообщение** с текстом или медиа,
которое будет показываться **перед вопросом**.

**Важно: этот бот не может создавать анонимные опросы. При прохождении теста
в группах, их участники всегда будут видеть, кто голосовал за какой вариант.**

BUTTONS:
  [0,0]: 'Создать вопрос'
```

Кнопка "Создать вопрос" — это deep-link на Telegram-клиент,
открывающий native poll creator. Из Telethon-userbot её **не нажимаем**, а
просто отправляем Poll через API.

### Шаг 5: отправили Poll (вопрос + варианты)

После отправки poll-а бот:

```
TEXT:
<no text>
```
(это **наш собственный poll**, отражённый ботом — игнорируем)

```
TEXT:
Пожалуйста, проголосуйте за правильный ответ.
```

Если poll был обычный (не quiz), нужно отдельно проголосовать через
`messages.SendVote` за индекс правильного ответа.

Если отправили сразу **quiz-poll** с `correct_answers` — этот шаг,
скорее всего, пропускается. Нужно подтвердить эмпирически.

### Шаг 6: голос за правильный ответ (только если был обычный poll)

После голоса бот:

```
TEXT:
В тесте «**История казахстана**» теперь 1 вопрос. Если Вы сделали ошибку в
вопросе, отправьте /undo.

Теперь отправьте следующий **вопрос**. Вы также можете отправить
**сообщение** с текстом или медиа, которое будет показываться **перед
вопросом**.

Когда тест будет готов, отправьте /done, чтобы опубликовать его.

BUTTONS:
  [0,0]: 'Создать вопрос'
```

### Шаг 7: либо ещё Poll → повтор шагов 5-6, либо `/done`

После `/done`:

```
TEXT:
Укажите ограничение времени для ответов. Бот будет отправлять в группу новый
вопрос, после того как истечет время на размышление.

Длинные промежутки рекомендуется использовать только если тест требует
сложных расчётов (например, решения математических задач). Для простых
тестов, как правило, достаточно 10-30 секунд на вопрос.

BUTTONS:
  [0,0]: '10 сек'
  [0,1]: '15 сек'
  [0,2]: '30 сек'
  [1,0]: '45 сек'
  [1,1]: '1 мин'
  [1,2]: '2 мин'
  [2,0]: '3 мин'
  [2,1]: '4 мин'
  [2,2]: '5 мин'
```

Дефолт для нас: **`30 сек`** (соответствует summary-строке `⏱ 30 сек`).

### Шаг 8: клик на кнопку с временем (например, "30 сек")

```
TEXT:
Показывать вопросы и варианты ответа в случайном порядке?

BUTTONS:
  [0,0]: 'Перемешать все'
  [0,1]: 'По порядку'
  [1,0]: 'Только вопросы'
  [1,1]: 'Только ответы'
```

Дефолт для нас: **`По порядку`** (если важен порядок вопросов из JSON).

### Шаг 9: клик на кнопку режима перемешивания

```

👍 Тест готов.
```
